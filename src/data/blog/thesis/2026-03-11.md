# Next Steps

Since we now have a properly formulated algorithm, and one in which we can defend mathematically, it is time to move forward to benchmarking. After some research on related work, I've found a great source of reference material for our benchmarking: PubLayNet (https://ar5iv.labs.arxiv.org/html/1908.07836).

This paper outlines the creation of a dataset for document layout analysis. Though that doesn't correspond 1:1 with our research intent, it's use of PDFMiner and the US NIH National Library of Medicine's Medical Article Records Groundtruth (MARG) dataset. Since these PDFs are `born digital`, we can get the 100% accurate text extraction from PDFMiner. Pairing that with the XML of the dataset we can retrieve the exact bounding boxes at the textline level with their corresponding text. It also gives a relative label benchmark, if we want to use that.

We can also create heuristics to group the textlines into bounding boxes. Since the source dataset is all academic papers we can measure the vertical gap between lines to determine if a line of text is within a new block. This XML may also provide us a ground-truth of the reading order. Though, I need to think on that more.

We can potentially use the following between PDFMiner, XML, and pdfplumber:
| Level | Source | How |
|--------|---------------------------------------------|----------------------------------------|
| region | PDFMiner textbox | direct |
| block | vertical gap detection within textbox | `gap > 1.5 × median_line_height` |
| line | PDFMiner textline | direct |
| word | PDFMiner `LTChar` groups or pdfplumber words | direct |
| order | NLM XML section sequence | via text matching |
